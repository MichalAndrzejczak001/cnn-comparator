❯ Jakich narzędzi użyto?

● Vitest — test runner natywny dla Vite, zamiennik Jest. Uruchamia testy przez npm test.

  @testing-library/react — renderuje komponenty React w jsdom i udostępnia render(), screen, waitFor(). Pozwala testować komponenty od strony użytkownika (przez tekst, role, atrybuty), nie przez wewnętrzną strukturę.

  @testing-library/user-event — symuluje realne interakcje użytkownika (user.click(), user.type()). Bardziej wiarygodne niż fireEvent, bo odtwarza sekwencję zdarzeń przeglądarki.

  @testing-library/jest-dom — dodatkowe matchery do expect: toBeInTheDocument(), toHaveTextContent(), toBeDisabled() itp.

  jsdom — środowisko DOM w Node.js, udostępnia document, localStorage, window. Vitest używa go zamiast prawdziwej przeglądarki.

  vi (z Vitest) — narzędzia do mockowania:
  - vi.mock('moduł') — podmienia cały moduł (użyte dla api/client i utils/csv w testach HistoryView)
  - vi.stubGlobal('Blob', class MockBlob {...}) — podmienia globalny konstruktor Blob klasą (wymóg Vitest 4 — arrow function nie może być konstruktorem)
  - vi.spyOn(document, 'createElement') — przechwytuje tworzenie elementu <a> w downloadCsv
  - vi.mocked(fetch).mock.calls — inspekcja argumentów wywołań zamockowanego fetch
